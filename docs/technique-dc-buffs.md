# Technique DC Buff Targets

Buffs that increase a technique's **save DC**, mirroring how PF1e handles spell
save DC. Two kinds of target are provided:

- **Technique DC (All)** — a global bonus applied to every technique.
- **Technique DC: `<discipline>`** — a per-discipline bonus, applied only to
  techniques of that discipline. One target per main discipline:
  Chakra Control, Fuinjutsu, Genjutsu, Ninjutsu, Taijutsu.

A technique's final save DC is:

```
DC = 10 + rank + abilityMod + saveOffset + dcBonus + globalDC + disciplineDC
```

where `globalDC` comes from the "All" buff and `disciplineDC` comes from the
buff matching the technique's own discipline. A Ninjutsu technique receives
`All + Ninjutsu`; a Taijutsu technique receives only `All` (plus its own
Taijutsu bonus if one exists).

## How it mirrors PF1e

PF1e (v11.11, compiled `pf1/pf1.js`) implements spell DC buffs as:

- A global buff target key `"dc"` → data path `system.attributes.spells.school.all.dc`.
- Per-school targets `"dc.school.<schoolId>"` → `system.attributes.spells.school.<schoolId>.dc`,
  generated by looping over `pf1.config.spellSchools`.
- `ItemAction#getDC` for a spell sums **both** the school-specific and the
  `all` bonus, plus a conditional `rollData.dcBonus`.

We use the technique's **discipline** as the analog of a spell school, and store
the bonuses in module flags instead of `system.*` (consistent with the rest of
the module, which never writes to core document data).

## Data model

Bonuses live in actor flags:

```
flags["naruto-d20"].techniqueDC.all.buffBonus            # global
flags["naruto-d20"].techniqueDC.{ckc,fui,gnj,nin,tai}.buffBonus   # per-discipline
```

These are write targets for PF1e's changes engine. They are reset to `0` in
`prepareBaseActorData` (so the engine writes onto a clean slate each prep) and
read on demand at roll time — no derived total is precomputed.

The per-discipline sub-key is the **skill key** (`ckc/fui/gnj/nin/tai`), the same
key used by learn checks. The mapping from a technique's `discipline` string to
that key is `DISCIPLINE_SKILL_MAP` in `scripts/data/skills.mjs`. Disciplines
without a learn skill (Hachimon Tonkou, Training) only ever receive the global
bonus.

## Buff target registration

Targets are declared once in `BUFF_TARGETS` (`scripts/flag-paths.mjs`), the
module's single source of truth for changes-engine integration:

```js
techDcAll: { label: "NarutoD20.BuffTargets.TechDc.All", path: techniqueDCBuffPath("all"), sort: 90010, category: "technique" },
techDcCkc: { label: "NarutoD20.BuffTargets.TechDc.ckc", path: techniqueDCBuffPath("ckc"), sort: 90011, category: "technique" },
// fui / gnj / nin / tai ...
```

Each entry carries an optional `category` (defaults to `"chakra"`).
`_registerBuffTargets` (`scripts/main.mjs`) registers a new
`technique` buff-target category and localizes labels via `game.i18n.localize`
(plain English labels of the older targets are not valid i18n keys, so they pass
through unchanged):

```js
CONFIG.PF1.buffTargetCategories.technique = { label: game.i18n.localize("NarutoD20.BuffTargets.Category") };
for (const [key, { label, sort, category = "chakra" }] of Object.entries(BUFF_TARGETS)) {
    CONFIG.PF1.buffTargets[key] = { label: game.i18n.localize(label), category, sort };
}
```

The `pf1GetChangeFlat` hook (`scripts/main.mjs`) needs no change — it already
maps any registered target to `BUFF_TARGETS[target].path`.

## DC computation

`scripts/data/technique-save-dc.mjs` patches
`pf1.components.ItemAction.prototype.getDC` for technique items and reads the
bonuses directly from actor flags (avoids depending on flags being present in
`rollData`):

```js
const ability  = resolveSkillAbility(this.item.actor, skillKey);
const ablMod   = this.item.actor?.system.abilities?.[ability]?.mod ?? 0;
// ...
const tdc      = this.item.actor?.flags?.["naruto-d20"]?.techniqueDC ?? {};
const globalDC = Number(tdc.all?.buffBonus) || 0;
const typeDC   = skillKey ? (Number(tdc[skillKey]?.buffBonus) || 0) : 0;

return 10 + rank + ablMod + offset + dcBonus + globalDC + typeDC;
```

`resolveSkillAbility(actor, skillKey)` (`scripts/data/skills.mjs`) is a shared
helper — `actor.system.skills[k].ability || NARUTO_SKILLS[k].ability || "int"` —
used by both `getDC` and `prepareDerivedActorData`, replacing what was
previously duplicated inline in both places.

| File | Change |
|---|---|
| `scripts/flag-paths.mjs` | `techniqueDCBuffPath` builder + 6 `BUFF_TARGETS` entries (category `technique`) |
| `scripts/main.mjs` | `_registerBuffTargets`: register `technique` category, localize labels, honor per-entry `category` |
| `scripts/data/derived-data.mjs` | Reset `techniqueDC.*.buffBonus` in `prepareBaseActorData`; use `resolveSkillAbility` |

No actor migration is needed: `prepareBaseActorData` seeds the `techniqueDC`
flag schema on every data prep.

## Manual verification

1. Reload the world (`F5`). Open a Buff item → **Changes** tab → add a change:
   the target picker shows a **Technique** category with "Technique DC (All)"
   and the five per-discipline targets (localized in pt-BR if the client is
   in pt-BR).
2. Create "Technique DC (All) +2" and "Technique DC: Ninjutsu +3", apply to an actor.
3. Use a **Ninjutsu** technique with a save action → DC rises by +5 (2 global + 3 nin)
   over `10 + rank + abilityMod`.
4. Use a **Taijutsu** technique → DC rises by only +2 (global only).
5. Remove/disable the buffs → DC returns to base, confirming the reset in
   `prepareBaseActorData`.
